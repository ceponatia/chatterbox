# Developer Documents Folder

This folder contains documents that are relevant to the development of the project. The [completed](./completed) subfolder contains plan and implementation documents that are complete and can be used for reference on past work.

Efforts should be made to keep these documents up to date and relevant to the current state of the project, with the exception of historical documents in the completed folder.

When a new plan or implementation document supercedes one in the completed folder, the new document should state this with how and why it supercedes the old document (e.g., code that it replaces, systems that are now deprecated and are being replaced, etc.).

The `tbd.md` document is a running idea list generated by the user that plans and implementation documents are generally built from. Do not edit this unless asked.

## Legend

- `PL` - Plan documents - May have implementation steps in them but are more broadly scoped than IM documents.
- `IM` - Implementation documents - Focused on implementing a specific plan or feature.
- `COMP` - Completed documents - Documents that are complete and can be used for reference on past work. This is a prefix to `PL` or `IM` documents.
- `DEP` - Deprecated documents - Documents that are no longer relevant and should not be used. This is a prefix to `PL` or `IM` documents.
